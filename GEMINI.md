# 프로젝트 컨텍스트

이 프로젝트는 **종단간 암호화(E2EE)를 지원하는 실시간 WebSocket 채팅 애플리케이션**입니다.

## 기술 스택
- **Frontend**: Next.js 16, React 19, TypeScript
- **Backend**: Custom Next.js Server, Socket.io, Supabase
- **암호화**: Web Crypto API (AES-GCM, PBKDF2)

## 아키텍처 패턴
- **DAO Pattern**: `src/dao/` - 데이터베이스 접근 로직
- **Model Pattern**: `src/models/` - 비즈니스 로직 및 도메인 모델
- **i18n**: 객체 접근 방식 (`t.auth.login` 형태)
- **선언적 라우팅**: 중앙화된 라우트 관리

### 중점 검토 사항
1. **보안**: 
   - 비밀번호 및 암호화 키가 클라이언트에서만 처리되는지 확인
   - `publicKey`를 사용한 사용자 인증이 올바르게 구현되었는지 확인
   
2. **타입 안정성**: 
   - TypeScript 타입이 명확하게 정의되어 있는지 확인
   - `any` 사용을 최소화하고 있는지 확인

3. **코드 스타일**:
   - ESLint 및 Prettier 규칙을 준수하고 있는지 확인
   - 일관된 코딩 컨벤션 유지

4. **성능**:
   - React Hook의 의존성 배열이 올바르게 설정되었는지 확인
   - 불필요한 리렌더링이 발생하지 않는지 확인

5. **아키텍처**:
   - DAO와 Model 계층 분리가 적절한지 확인
   - 비즈니스 로직이 적절한 위치에 있는지 확인
