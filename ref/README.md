# Ref (참고 자료)

## 개요
프로젝트 개발 과정에서 생성된 학습 자료, 작업 일지, 문서를 관리하는 디렉토리입니다.

## 디렉토리 구조

### daily/
일일 작업 기록을 저장합니다.

- **형식**: `YYYY-MM-DD-작업내용.md`
- **목적**: 매일의 개발 진행 상황, 해결한 문제, 배운 내용 기록
- **예시**: 
  - `2025-12-02-websocket-chat-fixes.md`: WebSocket 채팅 버그 수정 기록

### study/
기술 스터디 자료와 가이드 문서를 저장합니다.

- **e2e-encryption-guide.md**: End-to-End 암호화 구현 가이드
- **public-key-distribution.md**: 공개키 분배 방식 설명
- **socketio-guide.md**: Socket.io 사용법 및 예제

### docs/
프로젝트 관련 종합 문서를 저장합니다. (향후 추가 예정)

- 아키텍처 문서
- API 명세서
- 배포 가이드
- 트러블슈팅 가이드

## 사용 가이드

### daily/ 작성 규칙
매일 작업 종료 시 다음 내용을 포함하여 기록합니다:
- 작업한 내용 요약
- 발견한 버그와 해결 방법
- 새로 배운 기술이나 개념
- 다음 작업 계획

### study/ 작성 규칙
새로운 기술이나 개념을 학습할 때:
- 핵심 개념 설명
- 실제 코드 예제
- 참고 링크
- 프로젝트 적용 방법

### docs/ 작성 규칙 (향후)
공식 문서 작성 시:
- 명확하고 구조화된 형식
- 코드 예제 포함
- 목차 추가
- 버전 정보 명시

## 주의사항
- 이 디렉토리의 파일들은 프로덕션 빌드에 포함되지 않습니다
- 민감한 정보 (API 키, 비밀번호 등)는 절대 기록하지 않습니다
- GitHub에 모두 공개되므로 개인정보에 유의합니다
