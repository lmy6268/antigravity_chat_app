# 커밋 컨벤션

이 프로젝트는 한글 커밋 메시지를 사용하며, 다음과 같은 형식을 따릅니다.

## 커밋 메시지 형식

```
<타입>: <제목>

<본문>
```

### 타입 (Type)

- **feature**: 새로운 기능 추가
- **fix**: 버그 수정
- **refactor**: 코드 리팩토링 (기능 변경 없음)
- **style**: 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음)
- **docs**: 문서 수정
- **test**: 테스트 코드 추가 또는 수정
- **chore**: 빌드 설정, 패키지 매니저 설정 등
- **perf**: 성능 개선
- **security**: 보안 관련 수정

### 제목 (Subject)

- 50자 이내로 간결하게 작성
- 마침표를 붙이지 않음
- 명령형으로 작성 (예: "추가했음" ❌, "추가" ✅)
- 한글로 작성

### 본문 (Body)

- 제목과 본문 사이 한 줄 비우기
- 각 항목은 `-`로 시작
- 무엇을, 왜 변경했는지 설명
- 여러 변경사항이 있을 경우 그룹화하여 작성
- 한글로 작성

## 커밋 메시지 예시

### 예시 1: 기능 추가

```
feature: 종단간 암호화(E2EE) 개선 및 방 초대 기능 추가

- 방 생성자 자동 입장 기능 구현 (비밀번호 입력 불필요)
- 참여자별 암호화 키 관리 시스템 추가
- 방 초대 컴포넌트 및 훅 구현 (ChatRoomInvite, useRoomInvite)
- 사용자 검색 API 개선 (부분 일치 검색 지원)
- 다국어 지원 확장 (한국어/영어)
```

### 예시 2: 버그 수정

```
fix: 로그인 시 공개키 검증 오류 수정

- 공개키 형식 검증 로직 개선
- 에러 메시지 다국어 처리 추가
- 예외 처리 강화
```

### 예시 3: 리팩토링

```
refactor: 대시보드 컴포넌트 분리 및 구조 개선

- 탭 네비게이션 컴포넌트 분리
- 친구 관리 로직을 useFriends 훅으로 추출
- QR 코드 표시 컴포넌트 분리
- 코드 가독성 및 유지보수성 향상
```

## 워크플로우

1. 변경사항 확인

```bash
git status
```

2. 모든 변경사항 스테이징

```bash
git add -A
```

3. 커밋 메시지 작성 (위 컨벤션 참고)

```bash
git commit -m "<타입>: <제목>

<본문>"
```

4. 원격 저장소에 푸시

```bash
git push origin <브랜치명>
```

## 팁

- 커밋은 논리적 단위로 분리하여 작성
- 하나의 커밋에는 하나의 목적만 포함
- 커밋 메시지만 보고도 변경 내용을 파악할 수 있도록 작성
- 코드 리뷰어가 이해하기 쉽게 작성
